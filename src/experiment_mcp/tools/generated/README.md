# tools/generated/

Auto-generated MCP tools live here. Each module wraps one symbol
(class or function) from SweetPea / SweetBean and is regenerated by
the (future) `scripts/generate_tools.py` from
`generator/seeds.txt`.

Symmetric with `psyneulink-mcp/src/psyneulink_mcp/tools/generated/`.

## Conventions

- **Do not edit by hand.** Anything here is overwritten on the next
  regen. Behaviour overrides go in `tools/curated/` instead.
- The package's `__init__.py` is also regenerated; on a fresh clone it
  exposes an empty `ALL` tuple and a no-op `register_all`, so the
  server boots without any generated tools present.
- Generated tools register through `@captured_tool(mcp,
  layer="generated", ...)` from `experiment_mcp.feedback`, same as the
  curated layer, so runtime errors auto-capture into
  `feedback/pending/issues.jsonl`.
- Generated tool bodies import sweetpea / sweetbean directly (the
  generator only emits modules for symbols it actually found at
  introspection time, so PNL-style "import on demand" is not required
  here).

## Roadmap

The generator itself does not exist yet. Phase 2 t6e tracks landing
